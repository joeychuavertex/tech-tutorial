Agent = Model + Harness 
The harness is everything around the model except the model itself — and it's where the real work lives.
FEB2026

The year it got a name

The concept predates it — but the vocabulary arrived fast.
who

Mitchell Hashimoto — co-founder of HashiCorp, creator of Terraform — named the practice "Engineer the Harness" in a Feb 5 blog post.

then

Days later, OpenAI shipped a field report using the same term — and the word stuck across the industry.

root

"Harness" borrows from horse tack: the reins, saddle, and bit that steer raw power somewhere useful.

The model is the engine.
The harness is the car.

Raw, a model is brilliant but undirected — it can't touch a file, recall yesterday, or stop itself. The harness adds the steering, the memory, and the brakes. Same engine, suddenly it ships.

// the mechanic · the loop

It runs a loop.

Reason → Act → Observe → repeat. The harness owns every turn of it.

# the harness, not the model, runs this
while not task.done():
    step  = model.reason(context)
    if step.tool_call:
        result = harness.dispatch(step)
        context.add(result)   # observe
    context = harness.fit(context) # trim
// the mechanic · what's inside

Six parts, two jobs.

Guides steer it before it acts. Sensors catch it after.

Orchestration + tools

The loop and the dispatcher — what the agent can call, and when.

Memory + state

Persistence so the agent isn't blind at the start of every session.

Security + governance

Guardrails that block unsafe actions before they ever run.

Observability

Logs and metrics the agent can read to debug its own work.

Evals

Measurement — is the agent actually getting better, or just busy?

Instruction files

CLAUDE.md / AGENTS.md — the rules the agent reads before it starts.

Hashimoto's rule: when the agent errs, don't re-prompt — re-engineer the environment so that mistake becomes impossible.
